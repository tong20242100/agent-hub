# 小红书 MCP — 自动化发布

## 核心指令

通过 MCP 协议操作小红书发布和查询。**安全第一，频率控制。**

## 硬性限制（不可违反）

1. **独占登录**：运行时禁止在其他浏览器登录同一账号
2. **频率**：操作间隔必须 ≥ 30 秒
3. **日上限**：每日发布 ≤ 50 篇
4. **时间窗**：禁止 01:00-07:00 执行发布
5. **内容去重**：禁止直接上传未处理的原始图片

## 工具选择

| 工具 | 用途 | 示例 |
|------|------|------|
| `publish_note` | 发布笔记 | `publish_note title="..." content="..." images=["path"]` |
| `list_notes` | 查询笔记列表 | `list_notes limit=10` |
| `delete_note` | 删除笔记 | `delete_note note_id="..."` |
| `get_note_detail` | 获取笔记详情 | `get_note_detail note_id="..."` |

## 发布前检查清单

1. 检查系统时间，确认不在 01:00-07:00
2. 检查当日发布数量，确认未达 50 篇上限
3. 检查文案是否含诱导词（"微信"、"私聊"、"赚钱"），如有则重写
4. 确认图片已处理（非原始网络图片）
5. 确认距上次操作 ≥ 30 秒

## 发布后

1. 检查发布结果 `status: success`
2. 记录到 `workspace/xhs_log.txt`
3. 返回笔记 ID 和链接

## 错误处理

- 频率限制触发 → 等待 60 秒后重试
- 登录失效 → 提示重新认证
- 内容审核未通过 → 返回违规原因
