# Skill: Deep Researcher 深度研究任务指令

## 任务目标
执行基于实证证据的深度研究。通过多轮搜索、网页抓取及交叉验证，生成具备逻辑严密性和事实支撑的研究报告。

## 决策逻辑 (Decision Tree)
1. **初始分析**：
   - 提取关键词，划分研究维度（背景、技术细节、市场现状、竞争对比）。
2. **广度搜索**：
   - 使用 `agency-bin-search` 获取至少 5-10 个相关信源。
3. **深度验证**：
   - 针对核心论点，必须使用 `agency-bin-scrape` 抓取源网页内容。
   - **禁止** 仅依赖搜索片段（Snippets）。
4. **冲突处理**：
   - 发现相互矛盾的信息时，必须在报告中明确标注“信源冲突”，并根据信源权威度（如官方文档 vs 博客）进行加权评估。

## 输出规范 (Assets Alignment)
- 报告必须包含：核心结论、详细论证过程、参考来源列表。
- 格式：严格遵循 Markdown。

## 异常处理 (Resilience)
- **429 Rate Limit**：如果遇到频率限制，立即切换至 `lightpanda` 或 `scrapling-stealth` 尝试绕过。
- **内容缺失**：如果无法获取直接证据，必须尝试寻找“间接证据”并注明推论逻辑。
